01
Latar belakang masalah

Indonesia menghadapi paradoks keuangan yang menarik: tingkat inklusi keuangan sudah tinggi — hampir seluruh anak muda aktif menggunakan e-wallet seperti GoPay, OVO, dan DANA — namun pemahaman tentang cara mengelolanya masih rendah. Data OJK dan BPS melalui Survei Nasional Literasi dan Inklusi Keuangan (SNLIK) 2024 menunjukkan gap yang signifikan.

75%
Inklusi keuangan — sudah aktif pakai produk keuangan digital
65%
Literasi keuangan — paham cara mengelola keuangan dengan benar
56%
Literasi keuangan pelajar & mahasiswa — segmen paling rentan

Gap 10% antara inklusi dan literasi merepresentasikan jutaan orang yang aktif bertransaksi digital setiap hari, namun tidak memiliki pemahaman tentang alokasi keuangan yang sehat. Kelompok paling rentan adalah fresh worker — mereka yang baru pertama kali menerima gaji rutin dan belum memiliki kerangka untuk mengelolanya.

Tiga masalah utama

❓
Tidak tahu distribusi pengeluaran yang sehat
Fresh worker dengan gaji Rp 8–10 juta tidak tahu secara konkret berapa persen yang wajar untuk makan, tempat tinggal, hiburan, dan tabungan. Semua aplikasi yang ada hanya mencatat — tidak ada yang menjelaskan standar yang sehat dalam konteks Indonesia.
⏰
Tidak ada peringatan dini yang berarti
User baru sadar boros ketika tanggal 20 dan uang hampir habis. Tidak ada sistem yang memperingatkan bahwa satu kategori sudah mendekati batas jauh sebelum habis.
✍️
Input manual terlalu bergesekan
Setiap transaksi harus membuka app, memilih kategori, mengisi nominal — proses yang terasa seperti pekerjaan tambahan. Ini adalah alasan utama mayoritas pengguna meninggalkan expense tracker dalam minggu pertama.

Kompetitor yang ada (Money Lover, Wallet by BudgetBakers, Monefy) hanya menyelesaikan masalah pencatatan. Tidak ada yang menyelesaikan masalah pemahaman dan friksi input sekaligus — terutama dalam konteks lokal Indonesia.

02
Solusi

Fintrack mengadopsi mental model aplikasi gym dan menerapkannya ke manajemen keuangan pribadi. Seperti aplikasi gym yang menanyakan tujuan terlebih dahulu sebelum meresepkan program latihan, Fintrack menanyakan tujuan finansial sebelum mengalokasikan budget — bukan sekadar mencatat apa yang sudah terjadi.

03
Fitur
🎯
Onboarding goal-first MVP
6 pertanyaan sederhana tentang situasi dan tujuan finansial user. App generate alokasi budget otomatis yang dipersonalisasi — user tidak perlu tahu nama framework-nya. Expense items sudah diisi default, bisa diedit langsung sebelum selesai.
📸
Input foto struk → AI kategorisasi MVP
Foto nota atau struk belanja, AI membaca nominal dan merchant, lalu mengkategorikan secara otomatis. User hanya perlu konfirmasi. Total waktu: ~5 detik. Menyelesaikan masalah utama retensi: friksi input yang terlalu tinggi.
💪
Dashboard category fatigue MVP
Status tiap kategori pengeluaran: Fresh, Warning, atau Fatigued — persis seperti status otot di gym app. Disertai saran kontekstual yang terasa seperti coach, bukan akuntan. Update real-time setiap kali transaksi dicatat.
🛡
Dana darurat tracker MVP
Lacak progress dana darurat (target 3–6x pengeluaran bulanan) dengan visualisasi yang memotivasi. Estimasi waktu tercapai berdasarkan saving rate saat ini.
📋
Weekly summary narasi manusiawi MVP
Laporan mingguan dalam bahasa yang mudah dipahami, bukan hanya angka. Perbandingan dengan minggu sebelumnya dan satu saran konkret. Dikirim via email setiap Senin pagi.
⛓
Debt tracker — Snowball & Avalanche MVP
Untuk user Program Bebas Utang: daftar utang aktif, urutan pembayaran otomatis, estimasi bulan lunas, dan visualisasi "snowball" yang terbentuk setelah tiap utang lunas.
📈
Progressive saving rate v2
Sistem otomatis menyarankan kenaikan persentase tabungan setiap bulan kalau user konsisten — seperti progressive overload di gym.
🤖
Agent API / MCP Server (BYOA) v2
User yang punya AI agent sendiri (LangChain, Claude Desktop, dll) bisa generate access token dan sambungkan agent mereka ke data keuangan di Fintrack. Agent bisa query pengeluaran, catat transaksi, dan cek status budget via natural language.
04
User story

Onboarding

Sebagai fresh worker
Saya ingin menjawab beberapa pertanyaan tentang situasi keuangan saya, agar app bisa langsung memberikan rencana alokasi yang cocok tanpa saya harus set up manual dari nol.
✓ Onboarding selesai dalam < 3 menit · ✓ Budget items sudah diisi default · ✓ Bisa edit nominal sebelum konfirmasi
Sebagai fresh worker yang merantau
Saya ingin bisa menambahkan kosan dan KPR sekaligus sebagai pengeluaran wajib, meskipun keduanya melebihi 50% gaji, agar app tetap memberikan rencana yang realistis untuk situasi saya.
✓ App deteksi kalau kebutuhan > 50% dan berikan warning jujur · ✓ Program tetap dihasilkan dengan alokasi yang disesuaikan

Pencatatan transaksi

Sebagai pengguna yang sibuk
Saya ingin bisa mencatat pengeluaran hanya dengan foto struk, agar saya tidak perlu repot mengisi form manual setiap kali belanja.
✓ Foto → hasil AI muncul dalam < 3 detik · ✓ Bisa koreksi kategori sebelum simpan · ✓ Confidence score ditampilkan
Sebagai pengguna
Saya ingin langsung mendapat notifikasi status budget saat mencatat transaksi, agar saya tahu apakah pengeluaran saya masih aman tanpa harus buka halaman dashboard.
✓ Response POST /transactions mengandung fatigue_alert · ✓ Status ditampilkan inline di konfirmasi transaksi

Monitoring & fatigue

Sebagai pengguna
Saya ingin melihat status setiap kategori pengeluaran seperti indikator di gym app, agar saya tahu mana yang masih aman dan mana yang perlu dikurangi — tanpa harus menganalisis angka sendiri.
✓ Status Fresh/Warning/Fatigued jelas terlihat · ✓ Ada saran kontekstual per kategori · ✓ Update real-time
Sebagai pengguna
Saya ingin menerima ringkasan mingguan dalam bahasa yang mudah dimengerti setiap Senin, agar saya bisa evaluasi minggu lalu dan berencana untuk minggu depan.
✓ Narasi dalam Bahasa Indonesia · ✓ Ada perbandingan vs minggu lalu · ✓ Satu saran konkret yang actionable

Program Bebas Utang

Sebagai pengguna dengan utang CC Rp 20 juta
Saya ingin app memberitahu saya utang mana yang harus dilunasi duluan dan berapa yang harus dibayar setiap bulan, agar saya punya arah yang jelas tanpa harus menghitung sendiri.
✓ Urutan pembayaran otomatis (snowball/avalanche) · ✓ Estimasi bulan lunas tiap utang · ✓ Update setelah tiap pembayaran

Privacy & kepercayaan

Sebagai pengguna yang sensitif soal data finansial
Saya ingin tahu bahwa gaji saya tidak bisa dibaca oleh siapapun termasuk pemilik app, agar saya merasa aman memasukkan data keuangan yang sensitif.
✓ Income dienkripsi AES-256-GCM sebelum masuk DB · ✓ income_hint "Rp 8jt" yang ditampilkan di UI · ✓ Privacy Policy transparan di onboarding
Sebagai pengguna yang punya AI agent sendiri
Saya ingin bisa menghubungkan agent AI saya ke Fintrack, agar saya bisa tanya status keuangan saya langsung dari chat agent tanpa buka app.
✓ Generate access token dari settings · ✓ Token ditampilkan sekali saja · ✓ Bisa set scope read-only atau read-write

07
Database schema

Database: PostgreSQL via Supabase. Semua tabel dilindungi Row Level Security (RLS) — user hanya bisa akses data miliknya. Income dienkripsi AES-256-GCM di aplikasi sebelum masuk DB.

Tabel utama

-- ENUM TYPES CREATE TYPE expense_category_type AS ENUM ('fixed', 'variable', 'debt', 'want'); CREATE TYPE financial_program AS ENUM ('pondasi', 'bebas_utang', 'goal_chaser', 'tumbuh', 'seimbang'); CREATE TYPE fatigue_status AS ENUM ('fresh', 'warning', 'fatigued'); CREATE TYPE debt_method AS ENUM ('snowball', 'avalanche'); CREATE TYPE lifestyle_style AS ENUM ('easy', 'balanced', 'strict'); CREATE TYPE housing_type AS ENUM ('kosan', 'kpr', 'keluarga');
-- 1. USER PROFILES CREATE TABLE user_profiles ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL UNIQUE, -- FK ke auth.users Supabase income_encrypted TEXT, -- AES-256-GCM ciphertext income_hint VARCHAR(20), -- "Rp 8jt" untuk UI housing_type housing_type, lifestyle_style lifestyle_style, emergency_months SMALLINT DEFAULT 0, active_program financial_program, onboarding_done BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- 2. EXPENSE CATEGORIES (system default + custom user) CREATE TABLE expense_categories ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID, -- NULL = system default name VARCHAR(100) NOT NULL, icon VARCHAR(10), type expense_category_type NOT NULL, is_default BOOLEAN DEFAULT FALSE, is_active BOOLEAN DEFAULT TRUE, sort_order SMALLINT DEFAULT 0, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- 3. BUDGET PLANS (satu per user per bulan) CREATE TABLE budget_plans ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, period_year SMALLINT NOT NULL, period_month SMALLINT NOT NULL, total_income BIGINT NOT NULL, program financial_program NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), UNIQUE (user_id, period_year, period_month) ); -- 4. BUDGET ITEMS (alokasi per kategori dalam satu plan) CREATE TABLE budget_items ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), budget_plan_id UUID NOT NULL REFERENCES budget_plans(id) ON DELETE CASCADE, category_id UUID NOT NULL REFERENCES expense_categories(id), allocated_amount BIGINT NOT NULL, percentage NUMERIC(5,2), is_debt_focus BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), UNIQUE (budget_plan_id, category_id) ); -- 5. TRANSACTIONS CREATE TABLE transactions ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, budget_plan_id UUID REFERENCES budget_plans(id), category_id UUID NOT NULL REFERENCES expense_categories(id), amount BIGINT NOT NULL, -- plaintext, dilindungi RLS note TEXT, receipt_url TEXT, ai_categorized BOOLEAN DEFAULT FALSE, ai_confidence NUMERIC(3,2), transacted_at TIMESTAMPTZ NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- 6. DEBT ITEMS CREATE TABLE debt_items ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, category_id UUID REFERENCES expense_categories(id), name VARCHAR(100) NOT NULL, total_amount BIGINT NOT NULL, current_balance BIGINT NOT NULL, interest_rate NUMERIC(5,2) NOT NULL, min_payment BIGINT NOT NULL, method debt_method DEFAULT 'snowball', priority SMALLINT DEFAULT 1, is_active BOOLEAN DEFAULT TRUE, started_at DATE NOT NULL, target_paid_at DATE, paid_at DATE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- 7. GOALS CREATE TABLE goals ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, name VARCHAR(100) NOT NULL, icon VARCHAR(10), target_amount BIGINT NOT NULL, current_amount BIGINT DEFAULT 0, target_date DATE, is_completed BOOLEAN DEFAULT FALSE, is_primary BOOLEAN DEFAULT FALSE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- 8. WEEKLY REPORTS (cache hasil generate) CREATE TABLE weekly_reports ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, week_start DATE NOT NULL, week_end DATE NOT NULL, total_spent BIGINT NOT NULL DEFAULT 0, total_budget BIGINT NOT NULL DEFAULT 0, narrative TEXT, generated_at TIMESTAMPTZ, email_sent_at TIMESTAMPTZ, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(), UNIQUE (user_id, week_start) ); -- 9. API TOKENS (untuk BYOA) CREATE TABLE api_tokens ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), user_id UUID NOT NULL, name VARCHAR(100) NOT NULL, token_hash TEXT NOT NULL UNIQUE, -- bcrypt hash, tidak simpan plaintext token_hint VARCHAR(10), -- 4 char terakhir untuk display can_read BOOLEAN DEFAULT TRUE, can_write BOOLEAN DEFAULT FALSE, last_used_at TIMESTAMPTZ, expires_at TIMESTAMPTZ, is_active BOOLEAN DEFAULT TRUE, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() );
